## 1. Phase 1 - 필수 페이지 성공 기준

### 1.1 서비스 메인 페이지 (`/services`)

**시나리오**: 처음 방문한 사용자가 회사의 서비스를 이해하고 관심 있는 카테고리를 선택할 수 있다.

**Given**: 사용자가 `/services` 페이지에 접속
**When**: 페이지 로딩 완료
**Then**:
- [ ] Hero Section에 회사 슬로건 "아이디어를 현실로 만드는 파트너" 표시
- [ ] "개발 서비스" 섹션에 4개 카드 표시 (MVP, 풀스택, 디자인, 운영)
- [ ] 각 카드에 가격, 소요 기간, "자세히 보기" 버튼 포함
- [ ] "COMPASS 플랫폼" 섹션에 로드맵 타임라인 표시
- [ ] Navigator (현재), Cartographer/Captain/Harbor (2026 Q1 예정) 표시
- [ ] "왜 IDEA on Action인가?" 섹션에 4개 신뢰 요소 표시
- [ ] Lighthouse Performance 90+ 점

**Edge Cases**:
- [ ] 이미지 로딩 실패 시 placeholder 표시
- [ ] 느린 네트워크에서도 3초 이내 First Contentful Paint

### 1.2 MVP 개발 서비스 상세 (`/services/development/mvp`)

**시나리오 1**: 창업자가 MVP 개발 예산을 확인하고 적합한 패키지를 선택할 수 있다.

**Given**: 사용자가 `/services/development/mvp` 페이지에 접속
**When**: "가격 정책" 섹션까지 스크롤
**Then**:
- [ ] 3개 패키지 비교표 표시 (기본 ₩5M, 스탠다드 ₩8M, 프리미엄 ₩12M)
- [ ] 각 패키지의 핵심 기능 수, 기간, 무상 지원 기간 명시
- [ ] 결제 방식 (착수금 30%, 중간금 40%, 잔금 30%) 명시
- [ ] "상담 신청하기" CTA 버튼 클릭 시 `/work-with-us` 이동

**시나리오 2**: 개발자가 기술 스택을 확인하고 프로젝트 적합성을 판단할 수 있다.

**Given**: 사용자가 "기술 스택" 섹션 확인
**When**: Frontend/Backend/Database/Deployment 항목 읽기
**Then**:
- [ ] React, TypeScript, Vite, TailwindCSS (Frontend) 명시
- [ ] Supabase / Node.js + Express (Backend) 명시
- [ ] PostgreSQL (Supabase) (Database) 명시
- [ ] Vercel / AWS / Google Cloud (Deployment) 명시

**시나리오 3**: 사용자가 환불 정책을 확인하고 리스크를 이해할 수 있다.

**Given**: 사용자가 "환불 정책" 섹션 확인
**When**: 단계별 환불 규정 읽기
**Then**:
- [ ] 착수 전: 100% 환불
- [ ] 기획 단계 완료 전: 착수금 제외 환불
- [ ] 개발 단계 진행 중: 진행률에 따라 협의
- [ ] 개발 완료 후: 환불 불가 (계약 내용 미이행 시 제외)
- [ ] "자세한 내용은 환불 정책 페이지 참조" 링크 → `/refund` 이동

### 1.3 COMPASS Navigator 상세 (`/services/compass/navigator`)

**시나리오 1**: 프리랜서가 플랜을 비교하고 적합한 플랜을 선택할 수 있다.

**Given**: 사용자가 `/services/compass/navigator` 페이지의 "플랜 비교" 섹션 확인
**When**: Basic/Pro/Enterprise 비교표 읽기
**Then**:
- [ ] 3개 플랜의 월 이용료 명시 (₩29K, ₩99K, ₩299K)
- [ ] 12개 기능 항목별 차이 표시 (플랫폼 통합, 월 분석 건수, AI 분석 등)
- [ ] 연간 구독 시 20% 할인 안내
- [ ] 30일 무료 체험 (신규 가입자) 안내
- [ ] "무료 체험 시작하기" CTA 버튼 포함

**시나리오 2**: 사용자가 환불 정책을 확인하고 안전하게 구독을 시작할 수 있다.

**Given**: 사용자가 "환불 정책" 섹션 확인
**When**: 환불 규정 읽기
**Then**:
- [ ] 구독 시작 후 7일 이내: 100% 환불
- [ ] 7일 이후: 당월 환불 불가, 익월부터 해지
- [ ] 연간 구독: 사용하지 않은 개월 수 비례 환불
- [ ] 결제 방식: 신용카드 자동 결제, 언제든지 취소 가능

**시나리오 3**: 베타 테스터로 참여하여 무료로 Pro 플랜을 체험할 수 있다.

**Given**: 사용자가 "베타 테스터 모집" 섹션 확인
**When**: 혜택 내용 읽기
**Then**:
- [ ] 6개월간 Pro 플랜 무료 이용 (₩99K × 6 = ₩594K 상당)
- [ ] 신규 기능 우선 체험
- [ ] 피드백 제공 시 리워드
- [ ] "베타 테스터 신청하기" CTA 버튼 포함

## 3. Phase 3 - 최적화 성공 기준

### 3.1 SEO 최적화

**Given**: 모든 서비스 페이지가 배포됨
**When**: Google Search Console에서 크롤링
**Then**:
- [ ] 모든 페이지에 Open Graph 메타 태그 (title, description, image) 포함
- [ ] 구조화 데이터 (JSON-LD) 적용
  - Organization (회사 정보)
  - Service (서비스 정보)
  - Offer (가격 정보)
- [ ] sitemap.xml에 모든 서비스 페이지 포함
- [ ] robots.txt에 Allow 규칙 추가

### 3.2 모바일 반응형 체크

**Given**: 모바일 디바이스 (375px 너비)에서 접속
**When**: 모든 서비스 페이지 확인
**Then**:
- [ ] 카드 레이아웃 1열로 표시
- [ ] 텍스트 가독성 확보 (최소 16px)
- [ ] CTA 버튼 터치 영역 충분 (최소 44px)
- [ ] 가로 스크롤 발생하지 않음
- [ ] 이미지 반응형 (max-width: 100%)

### 3.3 접근성 개선

**Given**: 키보드만 사용하여 페이지 탐색
**When**: Tab 키로 네비게이션
**Then**:
- [ ] 모든 인터랙티브 요소에 포커스 표시
- [ ] Skip to content 링크 제공
- [ ] aria-label 적절히 사용
- [ ] 색상 대비 WCAG AA 충족 (최소 4.5:1)
- [ ] 스크린 리더로 읽을 때 의미 전달

### 3.4 로딩 속도 최적화

**Given**: Lighthouse 성능 테스트 실행
**When**: 각 페이지 측정
**Then**:
- [ ] First Contentful Paint < 1.8s
- [ ] Speed Index < 3.4s
- [ ] Largest Contentful Paint < 2.5s
- [ ] Time to Interactive < 3.8s
- [ ] Total Blocking Time < 200ms
- [ ] Cumulative Layout Shift < 0.1
- [ ] Performance Score 90+
