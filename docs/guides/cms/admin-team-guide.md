# AdminTeam 관리 가이드

## 📋 개요

### 목적
**AdminTeam 페이지**는 회사의 팀원 정보를 관리하는 관리자 전용 페이지입니다. 팀원의 프로필 사진, 이름, 직책, 소개, 스킬셋, 소셜 링크 등을 등록하고 관리할 수 있으며, 팀 구성 현황을 통계로 한눈에 파악할 수 있습니다. 공개 웹사이트의 "About" 페이지와 연동하여 업데이트된 팀원 정보를 자동으로 표시합니다.

### 주요 기능
✅ **팀원 데이터 관리**: 이름, 직책, 소개, 스킬셋, 소셜 링크 입력
✅ **아바타 이미지 관리**: 드래그 앤 드롭 이미지 업로드
✅ **검색 및 필터링**: 이름/직책 검색, 역할별 필터, 활성 상태 필터
✅ **CRUD 작업**: 팀원 추가, 수정, 삭제
✅ **빠른 상태 관리**: 활성/비활성 토글 스위치
✅ **우선순위 관리**: 표시 순서 결정
✅ **팀 통계**: 전체 인원, 활성 인원, 직책별 인원 카드

### 접근 권한
- **필요 권한**: Admin 이상 (editor, admin, super_admin 역할)
- **접근 경로**: `/admin/team`
- **권한 없을 경우**: 로그인 페이지로 리다이렉트

## 🎨 화면 구성

### 1️⃣ 목록 화면 (메인 페이지)

#### 상단 헤더
```
┌──────────────────────────────────────────┐
│ 팀원 관리                                      [새 팀원 추가]
│ 팀원 프로필을 관리합니다
└──────────────────────────────────────────┘
```

#### 📊 통계 카드 (4개)
- **Total**: 등록된 전체 팀원 수
- **Active**: 활성 상태 팀원 수
- **Founders**: "Founder" 직책 팀원 수
- **Developers**: "Developer" 직책 팀원 수

#### 🔍 검색 및 필터 영역
```
┌─────────────────────────┐ ┌─────────────┐ ┌─────────────┐
│ 🔍 이름 또는 역할 검색... │ │ 전체 역할   │ │ 전체 상태   │
└─────────────────────────┘ └─────────────┘ └─────────────┘
```

**검색**: 팀원 이름 또는 직책으로 실시간 검색
**역할 필터**: 전체, Founder, Developer, Designer, Community
**상태 필터**: 전체, 활성(Active), 비활성(Inactive)

#### 📋 팀원 목록 테이블
| 컬럼 | 설명 |
|------|------|
| **아바타** | 팀원 프로필 사진 (40x40px 원형) / 없으면 이니셜 배지 |
| **이름** | 팀원 이름 (굵은 글자) |
| **역할** | 직책 (배지 형식, 예: "Founder & CEO") |
| **스킬** | 스킬 뱃지 (최대 3개 표시, 초과시 "+N개" 표시) |
| **소셜** | 소셜 링크 아이콘 (GitHub, LinkedIn, Twitter, Website) |
| **우선순위** | 표시 순서 번호 (배지) |
| **활성** | 활성/비활성 토글 스위치 |
| **작업** | 수정(✏️), 삭제(🗑️) 버튼 |

#### 📭 빈 상태
검색 결과나 필터링 후 팀원이 없으면:
```
등록된 팀원이 없습니다
[첫 팀원 추가] 버튼
```

```
│   - 활성 상태 토글             │
│                                 │
│ ▶ 스킬 & 소셜 (접혀있음)       │
│   - 스킬셋 (Multi-Select)      │
│   - GitHub URL                 │
│   - LinkedIn URL               │
│   - Twitter URL                │

│                                 │

└─────────────────────────────────┘
```

## 🚀 기능 사용법

### 1️⃣ 팀원 목록 조회

#### 페이지 접속
1. 왼쪽 사이드바에서 **"팀원 관리"** 또는 `/admin/team` URL로 접속
2. 로딩 중이면 회전 아이콘이 표시되며, 완료 후 팀원 목록 테이블 표시

#### 통계 확인
상단의 4개 카드에서 팀 현황을 한눈에 파악:
- **Total**: 등록된 전체 팀원 수
- **Active**: 현재 활성 상태 팀원 수
- **Founders**: Founder/CEO 직책 인원
- **Developers**: Developer 직책 인원

#### 목록 확인
- 테이블의 각 행이 하나의 팀원 정보
- 아바타 이미지 클릭 시 새 탭에서 전체 이미지 보기 가능
- 우측 "작업" 칼럼의 버튼으로 수정/삭제 수행

---

### 🔍 검색 기능

#### 실시간 검색
1. 상단의 검색창에 **이름 또는 직책**을 입력합니다
   - 예: "홍", "Founder", "디자인"
2. **입력 동시에 자동 필터링** (300ms 디바운스 적용)
3. 대소문자 구분 없음
4. 부분 일치 지원 (검색어가 포함되면 결과에 표시)

#### 검색 예시
| 검색어 | 결과 |
|--------|------|
| `홍` | "홍길동" 팀원만 표시 |
| `founder` | 직책에 "founder"가 포함된 팀원들 표시 |
| `lead` | "Lead Developer", "Lead Designer" 등 표시 |

#### 검색 취소
- 검색창을 비워서 전체 목록으로 복원

---

### 🏷️ 필터 사용

#### 역할 필터
1. **"전체 역할"** 드롭다운 클릭
2. 선택 옵션:
   - **전체**: 모든 역할 표시
   - **Founder**: "Founder" 포함된 직책만 표시
   - **Developer**: "Developer" 포함된 직책만 표시
   - **Designer**: "Designer" 포함된 직책만 표시
   - **Community**: "Community" 포함된 직책만 표시

#### 상태 필터
1. **"전체 상태"** 드롭다운 클릭
2. 선택 옵션:
   - **전체**: 활성/비활성 모두 표시
   - **활성**: is_active = true인 팀원만
   - **비활성**: is_active = false인 팀원만

#### 필터 조합
검색 + 역할 필터 + 상태 필터를 **모두 동시 사용 가능**
- 예: "홍"으로 검색 + "Founder" 역할 필터 + "활성" 상태
- 세 조건을 모두 만족하는 팀원만 표시

#### 필터 초기화
- 각 드롭다운에서 "전체"를 선택하거나 검색창 비우기

### ➕ 팀원 추가

#### 단계 1️⃣: 팀원 추가 버튼 클릭
1. 페이지 우측 상단의 **"새 팀원 추가"** 버튼을 클릭합니다
2. 모달이 열리고 폼이 비워진 상태로 초기화됨
3. "새 팀원 추가" 제목이 표시됨

#### 단계 2️⃣: 기본 정보 입력 (Accordion - 기본 열림)
```
▼ 기본 정보
```

| 필드 | 필수 | 설명 |
|------|------|------|
| **이름** | ✅ | 2-100자, 예: "홍길동" |
| **직책** | ✅ | 2-100자, 예: "Founder & CEO", "Lead Developer" |
| **소개** | ❌ | 최대 2000자, Markdown 지원 |
| **우선순위** | ❌ | 0-9999 숫자 (높을수록 먼저 표시) |
| **활성 상태** | ❌ | 토글 스위치 (기본값: 활성) |

**입력 예시**:
```
이름: 홍길동
직책: Founder & CEO
소개:
  ## 프로필
  10년 경력의 풀스택 개발자입니다.
  - React, TypeScript 전문
  - 팀 리더십 경험
우선순위: 100
활성 상태: ON (활성)
```

#### 단계 3️⃣: 스킬 & 소셜 입력 (클릭하여 Accordion 열기)
```
▶ 스킬 & 소셜   ← 클릭하여 펼침
```

##### A. 스킬셋 입력
1. **"스킬셋" 드롭다운** 클릭
2. **사전 정의된 스킬** 중 선택:
   - React, TypeScript, JavaScript, Node.js, Python, Go, Rust
   - PostgreSQL, MongoDB, Redis, AWS, Docker, Kubernetes
   - Next.js, Vue.js, Svelte, Tailwind CSS, GraphQL, REST API
   - Supabase, Firebase, Figma, Adobe XD, UI/UX Design
   - Product Management, Community Management

3. **커스텀 스킬 추가** (목록에 없는 경우):
   - 입력창에 스킬명 타이핑
   - "Enter" 키를 누르거나 "생성" 버튼 클릭
   - 새로운 스킬이 선택 목록에 추가됨

4. **여러 스킬 선택**:
   - 각 스킬을 클릭하면 선택됨 (배지로 표시)
   - 최대 20개까지 선택 가능
   - 배지의 X를 클릭하여 제거

**입력 예시**:
```
스킬셋: React, TypeScript, Node.js, AWS, Docker
```

##### B. 소셜 링크 입력
1. **GitHub URL** (선택):
   - 예: `https://github.com/honggildong`

2. **LinkedIn URL** (선택):
   - 예: `https://linkedin.com/in/honggildong`

3. **Twitter URL** (선택):
   - 예: `https://twitter.com/honggildong`

4. **Website URL** (선택):
   - 예: `https://honggildong.dev`

**유효한 URL 형식**:
- ✅ `https://github.com/username`
- ✅ `http://example.com` (http도 가능)
- ❌ `github.com/username` (https:// 필수)
- ❌ `htp://example.com` (오타)

#### 단계 4️⃣: 아바타 업로드 (클릭하여 Accordion 열기)
```
▶ 아바타   ← 클릭하여 펼침
```

1. **아바타 업로드 영역** 클릭
2. **3가지 방법 선택**:
   - **파일 선택**: "파일 선택" 버튼 클릭
   - **드래그 & 드롭**: 이미지 파일을 영역에 드래그
   - **URL 입력**: 온라인 이미지 URL 직접 입력

3. **지원 포맷**: JPG, PNG, WebP, GIF
4. **권장 크기**: 300x300px 이상 정사각형 (프로필용)
5. **최대 크기**: 5MB

**업로드 후**:
- 미리보기 이미지 표시
- "제거" 버튼으로 삭제 가능
- 모달 닫기 시 자동 저장 (다시 열면 유지)

#### 단계 5️⃣: 저장
1. 모달 하단의 **"저장"** 버튼 클릭
2. **저장 중** 상태:
   - 버튼이 비활성화됨
   - 로딩 아이콘 표시
   - "저장 중..." 상태 표시
3. **완료 후**:
   - ✅ 성공: "팀원 생성 완료" 토스트 메시지
   - 모달 자동 닫힘
   - 목록에 새 팀원 추가됨
   - ❌ 실패: "팀원 생성 실패" 에러 메시지 + 오류 상세 정보

#### 유효성 검사 (실시간)
필드 입력 중 자동 검증:
- ❌ **이름 미입력**: "이름은 최소 2자 이상이어야 합니다"
- ❌ **직책 미입력**: "직책은 최소 2자 이상이어야 합니다"
- ❌ **잘못된 URL**: "유효하지 않은 URL입니다"
- ✅ 모든 필수 항목 입력 시 저장 버튼 활성화

### ✏️ 팀원 수정

#### 단계 1️⃣: 수정 모드 진입
1. 테이블에서 수정할 팀원 찾기
2. 우측 "작업" 칼럼의 **연필 아이콘 (✏️)** 버튼 클릭
3. 모달이 열리고 현재 정보가 모두 채워진 상태 표시
4. 제목: "팀원 수정"

#### 단계 2️⃣: 정보 수정
추가 방법과 동일하게 필드 수정:
- **기본 정보** Accordion: 이름, 직책, 소개, 우선순위, 활성 상태
- **스킬 & 소셜** Accordion: 스킬셋, 소셜 링크 (GitHub, LinkedIn, Twitter, Website)
- **아바타** Accordion: 아바타 이미지

#### 단계 3️⃣: 저장
1. 모달 하단의 **"수정"** 버튼 클릭 (추가와 다른 버튼명)
2. 저장 완료: **"팀원 수정 완료"** 토스트 메시지
3. 모달 자동 닫힘
4. 목록 즉시 갱신 (새로고침 불필요)

#### 수정 불가능 항목 (자동 관리)
| 필드 | 설명 |
|------|------|
| **ID** | 고유 식별자 (자동 생성) |
| **생성일** | 최초 생성 시간 (변경 불가) |
| **수정일** | 마지막 수정 시간 (자동 업데이트) |

### ⚡ 활성 상태 빠른 토글

테이블의 "활성" 칼럼에서 **스위치를 클릭**하여 즉시 상태 변경:

```
테이블 보기:
┌───────────┐
│ 활성      │ ← 스위치 ON (활성 상태)
│ 비활성    │ ← 스위치 OFF (비활성 상태)
└───────────┘
```

**동작**:
1. 스위치 클릭 → 활성/비활성 토글
2. **활성화**: "활성 상태 변경" 토스트 메시지
3. **비활성화**: "활성 상태 변경" 토스트 메시지
4. 모달 열 필요 없음 (즉시 처리)
5. 실패 시: "상태 변경 실패" + 오류 상세 정보

**활성/비활성의 의미**:
- ✅ **활성(Active)**: 공개 웹사이트의 About 페이지에 표시
- ❌ **비활성(Inactive)**: 관리자 페이지에서만 보임 (공개 웹사이트에 숨김)

### 🗑️ 팀원 삭제

#### 단계 1️⃣: 삭제 모드 진입
1. 테이블에서 삭제할 팀원 찾기
2. 우측 "작업" 칼럼의 **휴지통 아이콘 (🗑️)** 버튼 클릭
3. **삭제 확인 다이얼로그** 팝업 표시

#### 단계 2️⃣: 삭제 확인
```
┌─────────────────────────────────────┐
│ 팀원 삭제                        [X]  │
├─────────────────────────────────────┤
│ 정말로 이 팀원을 삭제하시겠습니까?  │
│ 이 작업은 되돌릴 수 없습니다.       │
├─────────────────────────────────────┤
│ [취소]                  [삭제] (빨강) │
└─────────────────────────────────────┘
```

#### 단계 3️⃣: 삭제 실행
1. **"삭제"** 버튼 클릭 (빨간색)
2. **삭제 진행 중**: 버튼 비활성화, 로딩 아이콘
3. **완료 후**:
   - ✅ 성공: **"팀원 삭제 완료"** 토스트 메시지
   - 다이얼로그 자동 닫힘
   - 목록에서 팀원 즉시 제거
   - ❌ 실패: "팀원 삭제 실패" 에러 메시지

#### 취소 방법
- "취소" 버튼 클릭
- 다이얼로그 닫기 (X 버튼)
- 다이얼로그 외부 클릭

#### ⚠️ 삭제 주의사항
- **복구 불가**: 삭제된 데이터는 영구적으로 삭제됨
- **권한 필수**: Admin 이상 권한 필요
- **연관 데이터**: 팀원 삭제 시 다른 데이터에는 영향 없음 (독립적 삭제)

## 🛡️ 주의사항

### 데이터 입력 시 주의
- **이름**: 최소 2자 이상, 공백만으로는 불가능
- **직책**: 최소 2자 이상, 자유로운 형식 가능 ("Co-Founder", "개발팀 팀장" 등)
- **소개**: Markdown 문법 지원하지만, HTML 태그는 보안상 제거됨
- **우선순위**: 더 높은 숫자가 먼저 표시 (예: 100 > 50 > 0)
- **URL**: 모두 https:// 또는 http://로 시작해야 함

### 아바타 이미지 관련
- **권장 크기**: 300×300px 이상의 정사각형 이미지
- **포맷**: JPG (권장), PNG, WebP, GIF
- **최대 크기**: 5MB
- **미리보기**: 업로드 후 모달 닫기 전에 제거 가능
- **없을 때**: 이름 첫 글자 + 배경색으로 표시

### 스킬셋 관련
- **사전정의 스킬**: 26개 기술 스킬 및 역량 제공
- **커스텀 추가**: 목록에 없는 스킬은 입력 후 Enter/생성 버튼으로 추가
- **최대 20개**: 20개 초과 입력 불가
- **중복 선택 불가**: 같은 스킬을 여러 번 선택할 수 없음

### 소셜 링크 관련
- **모두 선택 사항**: 4개 중 몇 개만 입력 가능
- **URL 형식**: https:// 또는 http://로 시작해야 함
- **유효성 검사**: 저장 시 URL 형식 자동 검증
- **옳은 예**:
  - `https://github.com/honggildong`
  - `https://linkedin.com/in/hong-gildong`
  - `https://twitter.com/honggildong`
  - `https://honggildong.dev`
- **잘못된 예**:
  - `github.com/honggildong` (https:// 누락)
  - `@honggildong` (소셜 핸들만 입력)
  - `htp://example.com` (오타)

### 권한 및 보안
- **접근 권한**: Admin 이상 (editor, admin, super_admin 역할)
- **RLS 정책**: Supabase 행 수준 보안(RLS)으로 데이터 보호
- **감시 기록**: 모든 CRUD 작업이 시스템 로그에 기록됨 (선택)

### 성능 관련
- **검색 지연**: 300ms 디바운스 적용 (타이핑 중 성능 최적화)
- **목록 로딩**: 팀원이 100명 이상일 경우 페이지네이션 고려
- **이미지 로딩**: 아바타는 CDN을 통해 캐시되어 빠른 로딩

## ❓ FAQ (자주 묻는 질문)

### Q1: 아바타 이미지 없이 팀원을 등록할 수 있나요?
**A**: ✅ 네, 가능합니다. 아바타는 선택 사항입니다. 아바타 없으면 **이름의 첫 글자가 배경색과 함께 표시**됩니다.
- 예: "홍길동" → 원형 배지에 "홍" 표시
- 예: "Alice Smith" → 원형 배지에 "A" 표시

---

### Q2: 스킬셋은 몇 개까지 입력 가능한가요?
**A**: ✅ **최대 20개**까지 입력 가능합니다. 그 이상 추가하면 "최대 20개까지만 추가 가능합니다" 오류 메시지가 표시됩니다.

**테이블 표시 규칙**:
- 첫 3개 스킬: 배지로 표시
- 4개 이상: "+N개" 표시 (예: "+5개")
- 모든 스킬 확인: 팀원 상세 페이지에서 전체 표시

---

### Q3: 스킬을 커스텀으로 추가할 수 있나요?
**A**: ✅ 네, 사전정의된 26개 스킬 외에도 커스텀 스킬 추가 가능합니다.

**방법**:
1. 스킬셋 입력창에서 새로운 스킬명 입력
2. **Enter 키** 또는 **"생성" 버튼** 클릭
3. 새 스킬이 바로 선택 목록에 추가됨
4. 예: "Flutter", "Kotlin", "AI/ML" 등 커스텀 기술 추가 가능

---

### Q4: 우선순위는 정렬에 어떻게 영향을 미치나요?
**A**: 우선순위(display_order)가 **높을수록 먼저 표시**됩니다.

**정렬 규칙**:
1. 우선순위 높은 순 (내림차순): 100 → 50 → 0 → -10
2. 같은 우선순위면 이름 오름차순 (A→Z)

**예시**:
```
팀원 A: 우선순위 100 → 1번째
팀원 B: 우선순위 100 → 2번째 (이름 순)
팀원 C: 우선순위 50  → 3번째
팀원 D: 우선순위 0   → 4번째
```

---

### Q5: 소셜 링크를 일부만 입력해도 되나요?
**A**: ✅ 네, 4개 중 원하는 것만 선택하여 입력 가능합니다.

**입력 규칙**:
- 모두 선택 사항 (필수 아님)
- 입력하지 않은 소셜은 테이블에 아이콘이 표시되지 않음
- 나중에 언제든 추가/수정 가능

**예시**:
- GitHub만 입력 → GitHub 아이콘만 표시
- GitHub + LinkedIn 입력 → 2개 아이콘 표시
- 모두 비움 → 소셜 아이콘 없음

---

### Q6: 소개에 Markdown 문법이 반영되나요?
**A**: ✅ 네, **Markdown 문법을 지원**합니다.

**지원 문법**:
- `## 제목` → 제목 크기 (H2)
- `**굵은 텍스트**` → 굵은 글자
- `*이탤릭*` → 기울어진 글자
- `[링크 텍스트](https://...)` → 클릭 가능한 링크
- `- 목록 항목` → 글머리 기호

**예시**:
```markdown
## 프로필
10년 경력의 **풀스택 개발자**입니다.

주요 기술:
- React, TypeScript
- Node.js, PostgreSQL
- AWS 배포 경험

[포트폴리오 보러가기](https://example.com)
```

### Q7: 비활성 상태(Inactive) 팀원은 어디에서 보이나요?
**A**: **관리자 페이지에서만** 볼 수 있습니다.

**표시 위치**:

- ❌ 공개 웹사이트: About 페이지에 **표시 안 됨**

**사용 예시**:
- 휴직 중인 팀원: 비활성 처리 → 나중에 활성 복구
- 팀을 떠난 팀원: 비활성으로 임시 보관 후 필요시 삭제
- 준비 중인 신입: 비활성으로 미리 등록

---

### Q8: 팀원을 실수로 삭제했을 때 복구 가능한가요?
**A**: ❌ **복구 불가능합니다.** 삭제된 데이터는 영구 삭제됩니다.

**주의사항**:
- 삭제 전 **확인 다이얼로그** 반드시 확인
- 데이터 실수는 **백업/재등록**으로 대응
- 중요 정보는 별도로 기록 권장

**예방 방법**:
- 먼저 "비활성" 처리로 숨김
- 며칠 후 확인 후 삭제 진행

### Q9: 아바타 이미지 URL을 업로드 중에 변경 가능한가요?
**A**: ✅ 네, 모달을 닫기 전까지 언제든 변경/제거 가능합니다.

**동작**:
- 업로드 후 "제거" 버튼으로 삭제
- 다른 이미지로 다시 업로드
- 모달 닫기 전에만 변경 가능 (저장 후는 수정 버튼으로 변경)

지금은 팀원을 하나씩 "새 팀원 추가" 버튼으로 등록해주세요.

## 🔧 문제 해결 (Troubleshooting)

### 1️⃣ 팀원이 저장되지 않을 때

**증상**: "저장" 버튼을 클릭해도 저장되지 않음

**해결 방법**:
1. ✅ **필수 필드 확인**:
   - **이름**: 입력됨? (2자 이상)
   - **직책**: 입력됨? (2자 이상)
   - 빨간색 오류 메시지 있는지 확인

2. ✅ **URL 형식 확인** (소셜 링크):
   - GitHub/LinkedIn/Twitter: `https://` 또는 `http://`로 시작?
   - 올바른 도메인?

3. ✅ **네트워크 연결 확인**:
   - 인터넷 연결 정상?
   - 브라우저 개발자 도구(F12) → Network 탭 확인
   - 오류 응답 있는지 확인

4. 🔄 **페이지 새로고침** (Ctrl+F5):
   - 캐시 삭제 후 다시 시도

5. 📞 **문제 지속 시**:
   - 개발팀에 오류 메시지 스크린샷 전달

---

### 2️⃣ 아바타 이미지가 표시되지 않을 때

**증상**: 이미지 업로드했는데 배지만 표시됨

**해결 방법**:
1. ✅ **URL 유효성 확인**:
   - 입력한 URL이 올바른가?
   - 브라우저에서 직접 URL 방문 가능?
   - 이미지 파일이 삭제되지 않았나?

2. ✅ **파일 크기 확인**:
   - 최대 5MB? (초과하면 오류)
   - 최소 50KB? (너무 작으면 표시 안 될 수 있음)

3. ✅ **파일 포맷 확인**:
   - JPG, PNG, WebP, GIF만 지원
   - BMP, TIFF 등 지원 안 함

4. 🔄 **페이지 새로고침**:
   - F5 새로고침 (캐시 삭제 안 됨)
   - Ctrl+F5 (브라우저 캐시 삭제)

5. 🗑️ **이미지 재업로드**:
   - 수정 버튼 클릭
   - 기존 이미지 제거
   - 다시 업로드

**참고**: Supabase 또는 CDN에서 이미지 로딩 지연이 있을 수 있습니다 (최대 1분).

---

### 3️⃣ 검색 결과가 없을 때

**증상**: 검색어를 입력했는데 결과 없음

**해결 방법**:
1. ✅ **검색어 확인**:
   - 띄어쓰기 있는가? (정확한 검색어 입력)
   - 특수문자 포함?
   - 대소문자 (관계없음)

2. ✅ **필터 상태 확인**:
   - 역할 필터: "전체" 선택?
   - 상태 필터: "전체" 선택?
   - 필터가 너무 좁으면 결과 0

3. ✅ **데이터 존재 확인**:
   - 팀원이 실제로 등록되어 있나? (테이블 비어있지 않나?)
   - 등록된 팀원의 이름/직책을 정확히 알고 있나?

4. 🔄 **검색창 초기화**:
   - 검색어 모두 지우기
   - Backspace로 하나씩 지우기
   - 필터도 "전체"로 초기화

5. 💾 **데이터 갱신**:
   - F5 새로고침
   - 페이지 새로고침 후 다시 시도

---

### 4️⃣ 필터가 작동하지 않을 때

**증상**: 필터 선택해도 목록이 안 바뀜

**해결 방법**:
1. ✅ **필터 옵션 확인**:
   - 역할 필터: "Founder", "Developer" 등에서 실제 직책이 있나?
   - 상태 필터: 활성 팀원이 실제로 있나?

2. ✅ **검색어 제거**:
   - 검색 + 필터 조합 시 둘 다 만족해야 함
   - 검색창 비우고 필터만 시도

3. 🔄 **필터 초기화**:
   - 역할 필터: "전체 역할" 선택
   - 상태 필터: "전체 상태" 선택

4. 🔄 **페이지 새로고침** (Ctrl+F5):
   - 캐시 삭제 후 다시 시도

---

### 5️⃣ "스킬 20개 초과" 오류가 발생할 때

**증상**: 스킬을 19개 선택했는데 "최대 20개"라는 오류 발생

**해결 방법**:
1. ✅ **선택된 스킬 확인**:
   - 모달의 선택 스킬 배지 개수 세기
   - 혹시 중복 선택은 없나?

2. 🗑️ **불필요한 스킬 제거**:
   - 배지의 X 버튼으로 하나 제거
   - "최대 20개까지만 가능합니다" 메시지 사라짐

3. ✅ **커스텀 스킬 확인**:
   - 자신이 생성한 커스텀 스킬도 개수에 포함됨
   - 중복된 커스텀 스킬 없는지 확인

4. 💡 **스킬 정리**:
   - 관련 스킬 통합 (예: "React", "React.js" → "React"만)
   - 정말 필요한 스킬만 선택

---

### 6️⃣ 모달이 열리지 않을 때

**증상**: "새 팀원 추가" 또는 "수정" 버튼을 클릭해도 모달 안 나타남

**해결 방법**:
1. ✅ **권한 확인**:
   - Admin 이상 권한이 있나? (로그아웃 후 다시 로그인)
   - 권한 없으면 버튼 자체가 비활성화됨

2. 🔄 **페이지 새로고침** (Ctrl+F5):
   - 캐시 문제로 UI 안 나타날 수 있음

3. 🗑️ **브라우저 개발자 도구 확인** (F12):
   - Console 탭에서 에러 메시지 있나?
   - 네트워크 요청 실패 없나?

4. 🔄 **다른 브라우저 시도**:
   - Chrome, Firefox, Safari 등 다른 브라우저에서 시도

5. 📞 **문제 지속 시**:
   - 개발팀에 브라우저 종류 및 버전 알리기

---

### 7️⃣ "유효하지 않은 URL" 오류가 발생할 때

**증상**: URL을 입력했는데 "유효한 URL을 입력하세요" 오류 발생

**해결 방법**:
1. ✅ **URL 형식 확인**:
   - `https://` 또는 `http://`로 시작?
   - 공백이 없나?
   - 올바른 도메인?

2. 📝 **올바른 형식 예시**:
   ```
   ✅ https://github.com/username
   ✅ https://linkedin.com/in/username
   ✅ http://example.com
   ❌ github.com/username (https:// 없음)
   ❌ https://github.com/username  (뒤에 공백)
   ❌ git hub.com/username (띄어쓰기)
   ```

3. 🔄 **URL 다시 입력**:
   - 필드 모두 지우기
   - 천천히 다시 입력
   - 복사-붙여넣기로 시도

4. ✅ **도메인 확인**:
   - 도메인 존재하나? (오타 없나?)
   - 실제 유효한 URL인가?
