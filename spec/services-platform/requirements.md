## 3. 페이지 구조 요구사항

### 3.1 필수 페이지 (Phase 1)
1. `/services` - 서비스 메인 페이지
   - Hero Section (회사 소개)
   - 개발 서비스 카테고리 (4개 카드)
   - COMPASS 플랫폼 카테고리 (로드맵 타임라인)
   - 신뢰 요소 (방법론, 기술 스택, 프로세스)

2. `/services/development/mvp` - MVP 개발 서비스 상세
   - 서비스 개요, 제공 내용 (4단계)
   - 기술 스택, 가격 정책 (3개 패키지)
   - 결제 방식, 진행 절차, 납품물
   - 환불 정책, FAQ

3. `/services/compass/navigator` - COMPASS Navigator 상세
   - 서비스 소개, 주요 기능 (4개)
   - 플랜 비교표 (Basic/Pro/Enterprise)
   - 가격 정책, 결제 방식, 환불 정책
   - 베타 테스터 모집 섹션

4. `/pricing` - 가격 안내 통합 페이지
   - 개발 서비스 가격표 (4개 서비스)
   - COMPASS 플랫폼 가격표 (4개 서비스)
   - 할인 혜택, 결제 수단

5. 법적 문서 페이지
   - `/terms` - 서비스 이용약관 (13조)
   - `/privacy` - 개인정보 처리방침 (10조)
   - `/refund` - 환불 정책 (9조)

### 3.2 확장 페이지 (Phase 2)
1. `/services/development/fullstack` - 풀스택 개발 서비스

## 4. 사용자 스토리

### 4.1 잠재 고객 (개발 서비스)
```
As a 스타트업 창업자,
I want to MVP 개발 서비스의 가격과 제공 내용을 명확히 확인하고,
So that 예산 범위 내에서 프로젝트 진행 가능 여부를 판단할 수 있다.
```

**Acceptance Criteria**:
- [ ] 3개 패키지의 가격, 기간, 제공 기능 수 비교표 제공
- [ ] 결제 방식 (착수금 30%, 중간금 40%, 잔금 30%) 명시
- [ ] 환불 정책 (단계별 환불 가능 금액) 명시
- [ ] 진행 절차 7단계 시각화
- [ ] FAQ 4개 이상 제공

### 4.2 프리랜서/에이전시 (COMPASS Navigator)
```
As a 프리랜서 개발자,
I want to COMPASS Navigator의 기능과 플랜을 비교하고,
So that 내 업무 스타일에 맞는 플랜을 선택하여 프로젝트 수주 기회를 늘릴 수 있다.
```

**Acceptance Criteria**:
- [ ] Basic/Pro/Enterprise 플랜 비교표 제공 (12개 기능 항목)
- [ ] 월 이용료 및 연간 구독 할인율 (20%) 명시
- [ ] 30일 무료 체험 안내
- [ ] 결제 방식 (신용카드 자동 결제) 명시
- [ ] 환불 정책 (7일 이내 100% 환불) 명시

### 4.3 법적 정보 확인
```
As a 서비스 이용자,
I want to 이용약관과 개인정보 처리방침을 쉽게 찾고 읽을 수 있기를,
So that 서비스 이용 전 권리와 의무를 명확히 이해할 수 있다.
```

**Acceptance Criteria**:
- [ ] Footer에 법적 문서 링크 3개 노출 (이용약관, 개인정보처리방침, 환불정책)
- [ ] 각 문서는 조항별로 구조화되어 가독성 확보
- [ ] 시행일자 명시
- [ ] 개인정보 보호책임자 연락처 명시
- [ ] 토스페이먼츠 제3자 제공 내역 명시

## 5. 비기능 요구사항

### 5.1 SEO
- 모든 페이지에 Open Graph 메타 태그 추가
- 구조화 데이터 (JSON-LD) 적용
- sitemap.xml에 서비스 페이지 포함
- robots.txt Allow 규칙 추가

### 5.2 접근성
- WCAG 2.1 AA 준수
- 키보드 네비게이션 지원
- 스크린 리더 호환

### 5.3 성능
- Lighthouse Performance 90+ 유지
- 이미지 최적화 (WebP, lazy loading)
- Code Splitting (서비스 페이지별 청크 분리)

### 5.4 반응형
- 모바일 퍼스트 디자인
- 브레이크포인트: 640px, 768px, 1024px, 1280px
- 모바일에서 카드 1열, 태블릿 2열, 데스크톱 3열
