# 컴포넌트 문서

이 디렉토리는 프로젝트의 주요 컴포넌트들에 대한 상세한 문서를 포함합니다.

## 📋 컴포넌트 목록

### 🎯 메인 컴포넌트

#### [Hero](./Hero.md)
메인 히어로 섹션 컴포넌트
- **용도**: 웹사이트의 첫인상을 담당하는 메인 섹션
- **주요 기능**: 브랜드 로고, 메인 메시지, CTA 버튼, 애니메이션 효과
- **접근성**: 키보드 네비게이션, 스크린 리더 지원, ARIA 라벨

#### [Features](./Features.md)
기능 소개 섹션 컴포넌트
- **용도**: 서비스의 주요 특징들을 카드 형태로 표시
- **주요 기능**: 반응형 그리드, 호버 효과, 인터랙티브 카드
- **접근성**: 시맨틱 HTML, 키보드 네비게이션, ARIA 라벨

#### [Services](./Services.md)
서비스 소개 섹션 컴포넌트
- **용도**: 제공하는 서비스들을 카드 형태로 표시
- **주요 기능**: 로딩/에러 상태, 반응형 그리드, 서비스 카드
- **접근성**: 상태별 접근성, 키보드 네비게이션, 스크린 리더 지원

### 🧭 네비게이션 컴포넌트

#### [Header](./Header.md)
네비게이션 헤더 컴포넌트
- **용도**: 사이트의 주요 네비게이션과 사용자 인증 상태 관리
- **주요 기능**: 브랜드 로고, 네비게이션 메뉴, 사용자 메뉴, 모바일 메뉴
- **접근성**: 키보드 네비게이션, 모바일 접근성, 스크린 리더 지원

#### [Footer](./Footer.md)
사이트 푸터 컴포넌트
- **용도**: 브랜드 정보, 네비게이션 링크, 소셜 미디어, 저작권 정보
- **주요 기능**: 소셜 링크, 네비게이션 링크, 저작권 정보
- **접근성**: 링크 접근성, 키보드 네비게이션, 스크린 리더 지원

## 📚 문서 가이드

### 컴포넌트 문서 구조
각 컴포넌트 문서는 다음 구조를 따릅니다:

1. **개요** - 컴포넌트의 목적과 역할
2. **Props** - 컴포넌트의 속성과 타입 정의
3. **사용 예시** - 기본 사용법과 커스터마이징 예시
4. **주요 기능** - 컴포넌트의 핵심 기능들
5. **접근성 고려사항** - 접근성 관련 사항
6. **성능 최적화** - 성능 관련 최적화 방법
7. **테스트** - 테스트 방법과 가이드
8. **브라우저 지원** - 지원 브라우저 목록
9. **관련 컴포넌트** - 관련된 다른 컴포넌트들
10. **변경 이력** - 버전별 변경 사항
11. **개발 팁** - 개발 시 유용한 팁들

### 문서 작성 가이드라인

#### 1. 코드 예시
```tsx
// 좋은 예시
<Component 
  prop1="value1" 
  prop2="value2" 
  className="custom-class"
/>

// 나쁜 예시
<Component prop1="value1" prop2="value2" className="custom-class"/>
```

#### 2. 접근성 고려사항
- ARIA 라벨 사용
- 키보드 네비게이션 지원
- 스크린 리더 호환성
- 색상 대비 준수

#### 3. 성능 최적화
- 불필요한 리렌더링 방지
- 메모이제이션 활용
- 이미지 최적화
- 번들 크기 최적화

## 🧪 테스트 가이드

### 단위 테스트
```bash
# 특정 컴포넌트 테스트
npm run test:unit ComponentName.test.tsx

# 모든 컴포넌트 테스트
npm run test:unit
```

### 접근성 테스트
```bash
# 접근성 테스트
npm run test:a11y ComponentName
```

### 시각적 테스트
```bash
# 시각적 회귀 테스트
npm run test:visual
```
