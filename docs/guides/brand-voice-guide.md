## 브랜드 정체성

### Who We Are

**IDEA on Action은 "커뮤니티형 프로덕트 스튜디오"입니다.**

- **정의**: 아이디어를 실험하고, 결과를 공유하며, 함께 성장하는 공간
- **차별점**: 단순한 개발사가 아닌, 사용자와 함께 만드는 혁신의 파트너
- **미션**: "생각을 멈추지 않고, 행동으로 옮기는 회사"

### Core Loop

```
아이디어 → 실험 → 결과 공유 → 참여 → 다음 아이디어
```

모든 콘텐츠는 이 순환 구조를 염두에 두고 작성합니다.

## 핵심 메시지

### 1. 함께 만드는 혁신

**Before (기술 중심)**:
> "120개의 커밋으로 완성한 고급 분석 대시보드"

**After (사용자 가치 중심)**:
> "매주 10건 이상의 업데이트로 사용자와 함께 완성한 분석 도구"

**Why**: 개발 과정보다 사용자 참여와 가치 창출을 강조

---

### 2. 투명한 과정, 실질적인 결과

**Before (프로세스 중심)**:
> "Phase 1-14를 거쳐 완성한 시스템"

**After (결과 중심)**:
> "6개월간의 실험으로 검증된, 5,000명이 사용하는 시스템"

**Why**: 추상적인 단계보다 구체적인 임팩트를 전달

---

### 3. 실험과 학습의 문화

**Before (완성품 강조)**:
> "완벽한 솔루션을 제공합니다"

**After (실험 강조)**:
> "빠르게 시도하고, 배우고, 개선합니다"

**Why**: 실패를 두려워하지 않는 혁신 문화 강조

## 톤앤매너

### Voice Personality

| 요소 | 설명 | 예시 |
|------|------|------|
| **협력적** | "우리", "함께", "같이" 사용 | "함께 만들어가요" |
| **격식 있으면서 친근** | 존댓말 + 이모지 절제 사용 | "프로젝트를 시작해보세요" |
| **구체적** | 추상적 표현 대신 숫자/예시 | "5,000명이 사용하는" |
| **긍정적** | 문제보다 해결에 집중 | "이렇게 개선했습니다" |
| **투명한** | 과정과 이유를 공유 | "이런 배경으로 결정했습니다" |

### Do's and Don'ts

#### Do's (권장)

✅ **협업 강조**
```markdown
"함께 만들어가는 프로젝트"
"커뮤니티와 함께 성장하는"
"당신의 아이디어를 기다립니다"
```

✅ **사용자 가치 중심**
```markdown
"매일 30분의 업무 시간을 절약"
"5,000명이 선택한 솔루션"
"월 평균 10건의 업데이트"
```

✅ **구체적인 표현**
```markdown
"지난 6개월간 120회 이상 개선"
"평균 응답 시간 2시간 이내"
"사용자 만족도 4.5/5.0"
```

#### Don'ts (지양)

❌ **기술 전문 용어**
```markdown
"Phase 1-14 완료"
"커밋 120개로 완성"
"KPI 달성률 95%"
"리스크 레벨 Low"
```

❌ **추상적인 표현**
```markdown
"최고의 솔루션"
"혁신적인 기술"
"완벽한 시스템"
```

❌ **일방적인 주장**
```markdown
"우리가 제공하는"
"최고의 품질을 보장"
"독점적인 기술"
```

## 페이지별 가이드

### 1. About (회사 소개)

**목표**: 회사 철학과 비전 전달

**톤**: 격식 있으면서 열정적

**필수 요소**:
- "우리는 ~입니다" (정체성)
- "이렇게 일합니다" (가치관)
- "함께 만들어가요" (참여 유도)

**예시**:

```markdown
# 우리는 생각을 멈추지 않는 회사입니다

IDEA on Action은 아이디어를 실험하고, 결과를 공유하며,
커뮤니티와 함께 성장하는 프로덕트 스튜디오입니다.

## 우리가 일하는 방식

- **투명하게 공유**: 모든 실험 과정을 공개합니다
- **빠르게 실행**: 완벽보다 빠른 시도를 선택합니다
- **함께 성장**: 커뮤니티 피드백으로 개선합니다
```

### 2. Roadmap (로드맵)

**목표**: 사용자 가치 중심의 계획 공유

**톤**: 협력적이고 구체적

**필수 요소**:
- 사용자 혜택 명시
- 예상 일정 공유
- 참여 방법 안내

**Before (기술 중심)**:
```markdown
## Q1 2025
- Phase 5 완료
- vendor chunk 최적화

## Q1 2025: 더 빠르고 안정적인 경험

- 페이지 로딩 속도 30% 개선
- 모바일 성능 최적화
- 사용자 피드백 반영 (215개 개선)

💡 **당신의 의견을 들려주세요**: [피드백 남기기]
```

### 3. Portfolio (포트폴리오)

**목표**: 문제 해결 스토리텔링

**톤**: 구체적이고 공감적

**구조**: 문제 → 해결 → 임팩트

**Before (기능 나열)**:
```markdown
## 프로젝트 A
- React + TypeScript
- Supabase 백엔드
- 6개월 소요
```

**After (스토리텔링)**:
```markdown
## 프로젝트 A: 중소기업 재고 관리 혁신

### 문제
중소 유통업체들이 엑셀로 재고를 관리하며
매월 평균 2시간의 수작업이 발생했습니다.

### 해결
실시간 재고 추적 시스템을 6개월간 개발하고,
50개 업체와 함께 3개월간 테스트했습니다.

### 임팩트
- 재고 관리 시간 70% 절감
- 재고 오차율 5% → 0.5%
- 50개 업체가 매일 사용 중

💡 **비슷한 문제가 있으신가요?**: [문의하기]
```

### 4. WorkWithUs (협업 제안)

**목표**: 참여 유도 및 신뢰 구축

**톤**: 초대하는, 환영하는

**필수 요소**:
- 명확한 참여 방법
- 기대 효과 제시
- 부담 없는 첫 단계

**Before (형식적)**:
```markdown
## 프로젝트 제안
아래 양식을 작성해 주세요.
[제안서 양식 다운로드]
```

**After (친근하고 구체적)**:
```markdown
## 당신의 아이디어를 함께 실현하세요

### 이런 분들과 함께합니다
- "좋은 아이디어는 있는데 어디서부터 시작할지 모르겠어요"
- "MVP를 빠르게 만들어 시장 반응을 보고 싶어요"
- "기술팀 없이도 프로덕트를 만들 수 있을까요?"

### 협업 과정
1. **30분 무료 상담**: 아이디어와 목표 공유
2. **2주 실험 설계**: 최소 비용으로 검증 계획
3. **함께 실행**: 투명하게 공유하며 개선

💡 **부담 없이 시작하세요**: [30분 상담 신청]
```

### 5. Lab (실험실)

**목표**: 실험 문화 공유 및 참여 유도

**톤**: 호기심 많고 배움에 열린

**필수 요소**:
- 실험 목적과 가설
- 결과 (성공/실패 모두)
- 배운 점

**예시**:
```markdown
## 실험 #42: AI 챗봇 응답 시간 개선

### 가설
GPT-4 Mini로 전환하면 응답 속도가 2배 빨라지고
사용자 만족도가 향상될 것이다.

### 실험 방법
- A/B 테스트 (GPT-4 vs GPT-4 Mini)
- 500명 사용자 대상
- 2주간 진행

### 결과
- 응답 속도: 3.2초 → 1.5초 (53% 개선)
- 만족도: 4.2 → 4.5 (7% 향상)
- **의외의 발견**: 짧은 질문에서 정확도 소폭 감소

### 배운 점
속도와 정확도의 트레이드오프를 고려해
질문 길이에 따라 모델을 동적 선택하도록 개선했습니다.

💡 **당신의 경험을 공유해주세요**: [댓글 남기기]
```

## 작성 체크리스트

### 콘텐츠 작성 전

- [ ] **대상 독자**: 누가 읽을 것인가?
  - 예: 기술자 / 비즈니스 의사결정자 / 일반 사용자
- [ ] **핵심 메시지**: 무엇을 전달할 것인가?
  - 예: "함께 만드는 혁신" / "투명한 과정" / "실질적인 결과"
- [ ] **행동 유도**: 어떤 행동을 기대하는가?
  - 예: 상담 신청 / 피드백 제공 / 커뮤니티 참여

### 작성 중

- [ ] **톤앤매너**: 격식 + 협력적인가?
- [ ] **협업 강조**: "함께", "같이", "우리" 사용했는가?
- [ ] **구체적 표현**: 숫자, 예시, 스토리 포함했는가?
- [ ] **사용자 가치**: 기술이 아닌 혜택을 강조했는가?
- [ ] **금지 용어**: Phase, KPI, 커밋 등 제거했는가?

### 작성 후

- [ ] **CTA 명확성**: 다음 행동이 명확한가?
- [ ] **가독성**: 짧은 문장, 명확한 구조인가?
- [ ] **일관성**: 브랜드 키워드 ("함께", "투명", "실험") 사용했는가?
- [ ] **감정**: 긍정적이고 초대하는 느낌인가?

### 예시 4: Lab 실험 보고서

**Before (기술 중심)**:
```markdown
## 실험 #42: GPT-4 → GPT-4 Mini 마이그레이션

목적: 응답 속도 개선 및 비용 절감

방법:
- A/B 테스트 (500명)
- 2주간 실행

결과:
- Latency: 3.2s → 1.5s
- Cost: -60%
- Accuracy: -2%

결론: 마이그레이션 완료
```

**After (학습 중심)**:
```markdown
## 실험 #42: AI 챗봇 응답 속도 2배 개선

### 가설
더 가벼운 AI 모델로 전환하면
사용자가 더 만족할 것이다.

### 실험
500명과 함께 2주간 테스트
기존 vs 새로운 모델 비교

### 배운 점
- 응답 속도 53% 개선 (3.2초 → 1.5초)
- 사용자 만족도 7% 향상
- **의외**: 짧은 질문에서 정확도 소폭 감소

### 개선
질문 길이에 따라 모델을 자동 선택하도록
시스템을 업그레이드했습니다.

💡 당신의 경험도 들려주세요 →
```
