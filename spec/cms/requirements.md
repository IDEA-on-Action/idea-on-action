**작성일**: 2025-11-15
**버전**: 1.0
**담당자**: Sinclair Seo

## 1. 프로젝트 개요

### 1.1 배경 및 목적
현재 IDEA on Action 웹사이트의 콘텐츠는 코드 레벨에서만 수정 가능하여, 비개발자가 콘텐츠를 관리하기 어려운 상황입니다. 관리자 모드 CMS를 구축하여:

- ✅ 개발자 개입 없이 콘텐츠 업데이트 가능
- ✅ 실시간 프로젝트 진행 상황 반영
- ✅ 포트폴리오 사례 추가/관리 간소화
- ✅ 실험실 아이디어 관리 효율화

### 1.2 범위 (Scope)
**In-Scope**:
- 로드맵, 포트폴리오, 실험실, 블로그, 팀원 정보 관리
- 관리자 인증 및 권한 관리 (3단계)
- 파일 업로드 (이미지, 문서)
- 리치 텍스트 에디터 (Markdown 지원)
- 실시간 프리뷰

**Out-of-Scope** (향후 확장):
- 다중 언어 콘텐츠 관리 (i18n CMS)
- 버전 히스토리 (Git-like 기능)
- 협업 기능 (동시 편집, 댓글)

## 2. 관리 대상 콘텐츠

### 2.1 로드맵 (Roadmap)
**사용자 스토리**:
> "관리자로서, 분기별 로드맵을 추가하고 진행 상황을 업데이트하여 팀과 고객에게 투명하게 공유하고 싶다."

**핵심 필드**:
- 제목, 설명, 카테고리 (서비스/플랫폼/내부)
- 상태 (계획/진행중/완료/보류)
- 진행률 (0-100%), 우선순위
- 시작일, 종료일, 태그

**성공 기준**:
- [ ] 로드맵 목록 페이지에서 분기별 필터링 가능
- [ ] 드래그 앤 드롭으로 우선순위 변경 (선택)
- [ ] 진행률을 프로그레스 바로 시각화
- [ ] 공개/비공개 설정 가능

### 2.2 포트폴리오 (Portfolio)
**사용자 스토리**:
> "관리자로서, 완료된 프로젝트를 포트폴리오로 등록하여 잠재 고객에게 성과를 보여주고 싶다."

**핵심 필드**:
- 프로젝트 제목, Slug, 간단한 설명
- 클라이언트 정보 (이름, 로고)
- 프로젝트 유형 (MVP/풀스택/디자인/운영)
- 썸네일, 추가 이미지 갤러리
- 기술 스택, 프로젝트 URL, GitHub URL
- 기간, 팀 규모, 시작/종료일
- 도전 과제, 해결 방법, 성과
- 고객 평가 (Testimonial)

**성공 기준**:
- [ ] 포트폴리오 목록 페이지에서 카드 그리드 형태로 표시
- [ ] 이미지 갤러리 드래그 앤 드롭 업로드
- [ ] 주요 프로젝트 고정 (featured) 기능
- [ ] Slug 자동 생성 (제목 → URL-friendly)

### 2.3 실험실 (Lab)
**사용자 스토리**:
> "관리자로서, 진행 중인 실험과 아이디어를 Lab에 등록하여 커뮤니티 참여를 유도하고 싶다."

**핵심 필드**:
- 제목, Slug, 부제목, 설명
- 콘텐츠 (Markdown 또는 리치 텍스트)
- 카테고리 (실험/아이디어/커뮤니티/연구)
- 상태 (탐색중/개발중/테스트중/완료/보관)
- 기술 스택, GitHub URL, 데모 URL
- 기여자 목록, 시작일, 태그

**성공 기준**:
- [ ] Lab 목록 페이지에서 상태별 필터링 가능
- [ ] GitHub 연동으로 커밋 히스토리 표시 (선택)
- [ ] 기여자 추가/제거 UI

### 2.4 블로그 (Blog)
**사용자 스토리**:
> "관리자로서, 기술 블로그 포스트를 작성하여 지식을 공유하고 SEO를 개선하고 싶다."

**핵심 필드**:
- 제목, Slug, 요약, 본문 (Markdown)
- 썸네일 이미지
- 작성자, 카테고리, 태그
- 예상 읽기 시간 (자동 계산)
- 조회수, 주요 포스트 여부
- SEO 제목, SEO 설명, 발행일

**성공 기준**:
- [ ] Markdown 에디터에서 실시간 미리보기
- [ ] 코드 블록 문법 하이라이팅
- [ ] 이미지 드래그 앤 드롭 삽입
- [ ] 초안 저장 기능

### 2.5 팀원 (Team)
**사용자 스토리**:
> "관리자로서, 팀원 정보를 추가하고 업데이트하여 About 페이지에 표시하고 싶다."

**핵심 필드**:
- 이름, 역할, 자기소개
- 아바타 이미지
- 스킬셋 (배열), 이메일
- 소셜 링크 (GitHub, LinkedIn, Twitter, 웹사이트)
- 표시 순서, 활성/비활성 상태

**성공 기준**:
- [ ] 드래그 앤 드롭으로 팀원 순서 변경
- [ ] 비활성화 시 공개 페이지에서 숨김

## 3. 관리자 권한 체계

### 3.1 역할 (Roles)
| 역할 | 설명 | 권한 |
|------|------|------|
| **Super Admin** | 최고 관리자 | 모든 기능 + 관리자 관리 |
| **Admin** | 일반 관리자 | 모든 콘텐츠 관리 |
| **Editor** | 편집자 | 콘텐츠 생성/수정만 가능 (삭제 불가) |

### 3.2 권한 매트릭스

| 기능 | Super Admin | Admin | Editor |
|------|-------------|-------|--------|
| 로드맵 생성/수정 | ✅ | ✅ | ✅ |
| 로드맵 삭제 | ✅ | ✅ | ❌ |
| 포트폴리오 생성/수정 | ✅ | ✅ | ✅ |
| 포트폴리오 삭제 | ✅ | ✅ | ❌ |
| 블로그 발행 | ✅ | ✅ | ✅ (초안만) |
| 블로그 삭제 | ✅ | ✅ | ❌ |
| 팀원 관리 | ✅ | ✅ | ❌ |
| 미디어 업로드 | ✅ | ✅ | ✅ |
| 미디어 삭제 | ✅ | ✅ | ❌ |
| 관리자 추가/제거 | ✅ | ❌ | ❌ |
| 설정 변경 | ✅ | ❌ | ❌ |

**사용자 스토리**:
> "Super Admin으로서, 신뢰할 수 있는 팀원에게 Editor 권한을 부여하여 콘텐츠 작성을 도와받되, 삭제 권한은 제한하고 싶다."

## 4. 파일 관리

### 4.1 미디어 라이브러리
**사용자 스토리**:
> "관리자로서, 업로드한 이미지와 파일을 한곳에서 관리하고 다른 콘텐츠에 재사용하고 싶다."

**기능 요구사항**:
- [ ] 파일 업로드 (드래그 앤 드롭 지원)
- [ ] 파일 타입 제한 (이미지: JPG/PNG/WebP/SVG, 문서: PDF)
- [ ] 파일 크기 제한 (최대 5MB)
- [ ] 그리드/리스트 뷰 전환
- [ ] 검색 및 필터링 (파일 타입, 업로드 날짜)
- [ ] 파일 선택 후 URL 복사
- [ ] 일괄 삭제

### 4.2 Supabase Storage 구조
```
buckets/
├── public/
│   ├── roadmap/        # 로드맵 이미지
│   ├── portfolio/      # 포트폴리오 이미지/파일
│   ├── lab/            # 실험실 이미지
│   ├── blog/           # 블로그 이미지
│   ├── team/           # 팀원 아바타
│   └── general/        # 기타 파일
└── private/
    └── admin/          # 관리자 전용 파일
```

## 5. 리치 텍스트 에디터

### 5.1 필수 기능
- [ ] 기본 서식 (굵게, 기울임, 밑줄, 취소선)
- [ ] 제목 (H1, H2, H3)
- [ ] 목록 (순서 있음, 순서 없음)
- [ ] 링크 삽입
- [ ] 이미지 삽입 (미디어 라이브러리 연동)
- [ ] 코드 블록 (문법 하이라이팅)
- [ ] 인용구
- [ ] Markdown 모드 전환

### 5.2 선택 기능 (향후 확장)
- [ ] 표 (Table) 삽입
- [ ] 유튜브 임베드
- [ ] 체크리스트

**사용자 스토리**:
> "편집자로서, Markdown 문법을 몰라도 블로그 포스트를 작성할 수 있는 WYSIWYG 에디터를 사용하고 싶다."

## 6. 검색 및 필터링

### 6.1 공통 기능
모든 콘텐츠 목록 페이지에서 지원:
- [ ] 검색 (제목, 설명)
- [ ] 필터링 (카테고리, 상태, 태그)
- [ ] 정렬 (최신순, 오래된 순, 제목순)
- [ ] 페이지네이션 (20개/페이지)

### 6.2 일괄 작업
- [ ] 체크박스로 다중 선택
- [ ] 일괄 삭제 (확인 대화상자)
- [ ] 일괄 발행/비공개

## 7. 실시간 기능 (선택)

### 7.1 Supabase Realtime
**사용자 스토리**:
> "관리자로서, 다른 관리자가 콘텐츠를 수정하면 실시간으로 목록에 반영되어 충돌을 방지하고 싶다."

**기능 요구사항** (Phase 2):
- [ ] 실시간 목록 업데이트
- [ ] 편집 중 표시 (다른 사용자가 편집 중일 때)
- [ ] 자동 저장 (30초마다)

## 8. SEO 최적화

### 8.1 공개 페이지 SEO
모든 공개 콘텐츠 페이지에서 지원:
- [ ] 메타 제목, 메타 설명 커스터마이징
- [ ] Open Graph 태그 (소셜 공유)
- [ ] Canonical URL
- [ ] Structured Data (JSON-LD)

### 8.2 Sitemap 자동 생성
- [ ] 콘텐츠 발행 시 sitemap.xml 자동 업데이트
- [ ] robots.txt 관리

## 9. 모바일 대응

### 9.1 반응형 디자인
- [ ] 모바일 (320px~767px): 1열 레이아웃
- [ ] 태블릿 (768px~1023px): 2열 레이아웃
- [ ] 데스크탑 (1024px+): 사이드바 + 메인 영역

### 9.2 모바일 전용 기능
- [ ] 햄버거 메뉴 (사이드바 토글)
- [ ] 터치 제스처 지원 (스와이프)

## 10. 완료 기준 (Definition of Done)

### Phase 1: 기반 구축 (2주)
- [ ] 관리자 인증 시스템 구축 (이메일/비밀번호 로그인)
- [ ] Supabase RLS 정책 설정 (admins, roadmap_items, portfolio_items 등)
- [ ] Admin Layout 구조 (Header, Sidebar, Breadcrumb)
- [ ] 공통 컴포넌트 (Button, Input, Table, Modal, Toast)

### Phase 2: 핵심 기능 (3주)
- [ ] 로드맵 CRUD (목록, 생성, 수정, 삭제)
- [ ] 포트폴리오 CRUD
- [ ] 실험실 CRUD
- [ ] 블로그 CRUD
- [ ] 리치 텍스트 에디터 통합

### Phase 3: 고급 기능 (2주)
- [ ] 미디어 라이브러리 구현
- [ ] 팀원 관리
- [ ] 카테고리/태그 관리
- [ ] SEO 설정 UI

### Phase 4: 최적화 및 테스트 (1주)
- [ ] 성능 최적화 (React Query 캐싱, 이미지 최적화)
- [ ] E2E 테스트 20+개
- [ ] 사용자 테스트 및 피드백 반영

## 11. 비기능 요구사항

### 11.1 성능
- [ ] 페이지 로딩 시간 < 2초
- [ ] 이미지 리사이징 (썸네일 자동 생성)
- [ ] Lazy Loading (목록 페이지)

### 11.2 보안
- [ ] HTTPS 전용
- [ ] CSRF 토큰 검증
- [ ] XSS 방지 (HTML 새니타이징)
- [ ] 파일 업로드 검증 (MIME 타입, 확장자, 크기)

### 11.3 접근성
- [ ] WCAG 2.1 AA 준수
- [ ] 키보드 네비게이션 지원
- [ ] ARIA 속성 추가
