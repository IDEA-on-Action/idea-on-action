# CMS 관리자 모드 성공 기준

**작성일**: 2025-11-15
**버전**: 1.0
**담당자**: Sinclair Seo

---

## 1. 로드맵 관리

### AC-01: 로드맵 목록 조회
**Given**: 관리자가 로그인한 상태에서
**When**: `/admin/roadmap` 페이지에 접근하면
**Then**:
- [ ] 로드맵 항목이 테이블 형태로 표시됨
- [ ] 각 항목은 제목, 카테고리, 상태, 진행률, 시작일을 포함
- [ ] 필터링 UI (카테고리, 상태 선택)가 표시됨
- [ ] "새로 만들기" 버튼이 표시됨
- [ ] 페이지네이션 (20개/페이지)이 동작함

### AC-02: 로드맵 생성
**Given**: 관리자가 로드맵 목록 페이지에서
**When**: "새로 만들기" 버튼을 클릭하고 폼을 작성하면
**Then**:
- [ ] 제목 필드는 필수 (200자 제한)
- [ ] 카테고리 선택 (서비스/플랫폼/내부)
- [ ] 상태 선택 (계획/진행중/완료/보류)
- [ ] 진행률 슬라이더 (0-100%)
- [ ] 시작일, 종료일 선택 (DatePicker)
- [ ] 태그 입력 (다중 선택)
- [ ] 공개/비공개 체크박스
- [ ] 저장 성공 시 목록 페이지로 리다이렉트
- [ ] 성공 토스트 메시지 표시

### AC-03: 로드맵 수정
**Given**: 관리자가 로드맵 목록에서 특정 항목을 선택하면
**When**: "편집" 버튼을 클릭하고 수정하면
**Then**:
- [ ] 기존 데이터가 폼에 미리 채워짐
- [ ] 수정 후 저장 시 변경사항이 반영됨
- [ ] 낙관적 업데이트 (Optimistic Update) 적용
- [ ] 성공 토스트 메시지 표시

### AC-04: 로드맵 삭제
**Given**: Admin 이상의 권한을 가진 관리자가
**When**: 로드맵 항목의 "삭제" 버튼을 클릭하면
**Then**:
- [ ] 확인 대화상자 표시 ("정말 삭제하시겠습니까?")
- [ ] "예" 클릭 시 항목이 삭제됨
- [ ] 목록에서 즉시 제거됨
- [ ] 성공 토스트 메시지 표시

**And**: Editor 권한 사용자는
- [ ] "삭제" 버튼이 비활성화됨
- [ ] 클릭 시 "권한이 없습니다" 메시지 표시

## 2. 포트폴리오 관리

### AC-05: 포트폴리오 목록 조회
**Given**: 관리자가 로그인한 상태에서
**When**: `/admin/portfolio` 페이지에 접근하면
**Then**:
- [ ] 포트폴리오 항목이 카드 그리드 형태로 표시됨
- [ ] 각 카드는 썸네일, 제목, 프로젝트 유형, 상태를 포함
- [ ] 필터링 UI (프로젝트 유형, 공개/비공개)가 표시됨
- [ ] 검색 입력 필드가 표시됨
- [ ] "새로 만들기" 버튼이 표시됨

### AC-06: 포트폴리오 생성
**Given**: 관리자가 포트폴리오 생성 페이지에서
**When**: 폼을 작성하고 저장하면
**Then**:
- [ ] 제목 필드는 필수
- [ ] Slug는 제목에서 자동 생성 (예: "ABC Project" → "abc-project")
- [ ] Slug 중복 시 에러 메시지 표시
- [ ] 프로젝트 유형 라디오 버튼 (MVP/풀스택/디자인/운영)
- [ ] 기술 스택 태그 입력 (최대 20개)
- [ ] 클라이언트 로고 업로드 (5MB 제한, JPG/PNG/WebP)
- [ ] 추가 이미지 업로드 (다중, 드래그 앤 드롭)
- [ ] 주요 프로젝트 체크박스 (featured)
- [ ] 저장 성공 시 상세 페이지로 리다이렉트

### AC-07: 이미지 갤러리 관리
**Given**: 관리자가 포트폴리오 편집 페이지에서
**When**: 이미지 갤러리 섹션에 파일을 드래그 앤 드롭하면
**Then**:
- [ ] 업로드 진행률 표시 (프로그레스 바)
- [ ] 업로드 완료 시 썸네일 미리보기 표시
- [ ] 각 이미지에 "삭제" 버튼 표시
- [ ] 이미지 순서 변경 가능 (드래그 앤 드롭, 선택)
- [ ] 5MB 초과 시 에러 메시지 표시
- [ ] 허용되지 않은 파일 타입 시 에러 메시지 표시

## 3. 블로그 관리

### AC-08: 블로그 포스트 작성
**Given**: 관리자가 블로그 작성 페이지에서
**When**: Markdown 에디터로 글을 작성하면
**Then**:
- [ ] 제목, Slug, 요약, 본문 입력 필드 표시
- [ ] Markdown 에디터에서 실시간 미리보기 제공
- [ ] 코드 블록 문법 하이라이팅 (JavaScript, Python, TypeScript 등)
- [ ] 이미지 드래그 앤 드롭으로 삽입
- [ ] 이미지 삽입 시 Markdown 문법 자동 생성 (`![alt](url)`)
- [ ] 초안 저장 버튼 (published: false)
- [ ] 발행 버튼 (published: true)
- [ ] 예상 읽기 시간 자동 계산 (본문 길이 기준)

### AC-09: 블로그 카테고리 관리
**Given**: 관리자가 카테고리 관리 페이지에서
**When**: 새 카테고리를 추가하면
**Then**:
- [ ] 카테고리 이름, Slug, 설명 입력
- [ ] 색상 선택 (hex color picker)
- [ ] 아이콘 선택 (lucide-react 아이콘 목록)
- [ ] 저장 후 카테고리 목록에 추가됨
- [ ] 블로그 포스트 작성 시 카테고리 선택 가능

### AC-10: 실험실 프로젝트 목록 조회
**Given**: 관리자가 로그인한 상태에서
**When**: `/admin/lab` 페이지에 접근하면
**Then**:
- [ ] 실험실 프로젝트가 카드 형태로 표시됨
- [ ] 각 카드는 제목, 카테고리, 상태, 기여자 수를 포함
- [ ] 상태별 필터링 (탐색중/개발중/테스트중/완료/보관)
- [ ] 카테고리별 필터링 (실험/아이디어/커뮤니티/연구)

### AC-11: 기여자 관리
**Given**: 관리자가 실험실 프로젝트 편집 페이지에서
**When**: 기여자 섹션에서 추가 버튼을 클릭하면
**Then**:
- [ ] 기여자 검색 입력 필드 표시
- [ ] 자동 완성으로 등록된 사용자 목록 표시
- [ ] 선택 시 기여자 목록에 추가됨
- [ ] 각 기여자에 "제거" 버튼 표시
- [ ] 저장 시 기여자 정보가 배열로 저장됨

## 5. 팀원 관리

### AC-12: 팀원 목록 조회
**Given**: 관리자가 로그인한 상태에서
**When**: `/admin/team` 페이지에 접근하면
**Then**:
- [ ] 팀원 목록이 테이블 형태로 표시됨
- [ ] 각 행은 아바타, 이름, 역할, 스킬셋, 상태를 포함
- [ ] 드래그 앤 드롭으로 순서 변경 가능
- [ ] 순서 변경 시 자동 저장 (낙관적 업데이트)

### AC-13: 팀원 추가
**Given**: 관리자가 팀원 추가 페이지에서
**When**: 팀원 정보를 입력하고 저장하면
**Then**:
- [ ] 이름, 역할 필드는 필수
- [ ] 아바타 이미지 업로드 (원형 크롭 미리보기)
- [ ] 스킬셋 태그 입력 (다중 선택)
- [ ] 소셜 링크 입력 (GitHub, LinkedIn, Twitter, 웹사이트)
- [ ] URL 형식 검증 (https:// 필수)
- [ ] 활성/비활성 토글 스위치
- [ ] 저장 후 팀원 목록에 추가됨

## 6. 미디어 라이브러리

### AC-14: 미디어 라이브러리 조회
**Given**: 관리자가 로그인한 상태에서
**When**: `/admin/media` 페이지에 접근하면
**Then**:
- [ ] 업로드된 파일이 그리드 형태로 표시됨
- [ ] 각 카드는 썸네일, 파일명, 크기, 업로드 날짜를 포함
- [ ] 그리드 뷰 / 리스트 뷰 전환 버튼
- [ ] 파일 타입 필터 (전체/이미지/문서)
- [ ] 검색 입력 필드 (파일명 기준)

### AC-15: 파일 업로드
**Given**: 관리자가 미디어 라이브러리에서
**When**: "파일 업로드" 버튼을 클릭하거나 드래그 앤 드롭하면
**Then**:
- [ ] 파일 선택 대화상자 표시
- [ ] 다중 파일 선택 가능
- [ ] 각 파일에 대해 업로드 진행률 표시
- [ ] 5MB 초과 파일은 에러 메시지 표시
- [ ] 허용되지 않은 파일 타입은 에러 메시지 표시
- [ ] 업로드 완료 시 그리드에 추가됨

### AC-16: 파일 URL 복사
**Given**: 관리자가 미디어 라이브러리에서 파일을 선택하면
**When**: "URL 복사" 버튼을 클릭하면
**Then**:
- [ ] 파일의 Public URL이 클립보드에 복사됨
- [ ] 성공 토스트 메시지 표시 ("URL이 복사되었습니다")

## 7. 권한 관리

### AC-17: 관리자 목록 조회 (Super Admin만)
**Given**: Super Admin이 로그인한 상태에서
**When**: `/admin/settings/admins` 페이지에 접근하면
**Then**:
- [ ] 관리자 목록이 테이블 형태로 표시됨
- [ ] 각 행은 이메일, 역할, 생성일을 포함
- [ ] "관리자 추가" 버튼 표시

**And**: Admin 또는 Editor가 접근하면
- [ ] 403 Forbidden 페이지 표시
- [ ] "권한이 없습니다" 메시지 표시

### AC-18: 관리자 추가 (Super Admin만)
**Given**: Super Admin이 관리자 추가 페이지에서
**When**: 사용자 이메일과 역할을 선택하고 저장하면
**Then**:
- [ ] 이메일 형식 검증
- [ ] 이미 관리자인 경우 에러 메시지 표시
- [ ] 역할 선택 (Super Admin/Admin/Editor)
- [ ] 저장 후 해당 사용자는 관리자 페이지 접근 가능
- [ ] 이메일 알림 발송 (선택)

## 8. 성능 및 UX

### AC-19: 로딩 상태 표시
**Given**: 관리자가 데이터를 조회할 때
**When**: 네트워크 요청이 진행 중이면
**Then**:
- [ ] 로딩 스피너 또는 스켈레톤 UI 표시
- [ ] 버튼은 비활성화 상태로 전환
- [ ] 로딩 완료 후 데이터 표시

### AC-20: 에러 처리
**Given**: 관리자가 데이터를 저장할 때
**When**: 네트워크 에러 또는 서버 에러가 발생하면
**Then**:
- [ ] 에러 토스트 메시지 표시 (빨간색)
- [ ] 에러 메시지는 구체적 (예: "제목은 필수입니다")
- [ ] 폼은 초기화되지 않음 (입력값 유지)
- [ ] 재시도 버튼 표시 (선택)

### AC-21: 페이지네이션
**Given**: 관리자가 목록 페이지를 조회할 때
**When**: 데이터가 20개를 초과하면
**Then**:
- [ ] 페이지네이션 UI 표시 (하단)
- [ ] 현재 페이지 번호 하이라이트
- [ ] 이전/다음 버튼 표시
- [ ] 페이지 번호 클릭 시 해당 페이지로 이동
- [ ] URL 쿼리 파라미터에 페이지 번호 포함 (`?page=2`)

## 9. SEO 및 공개 페이지 연동

### AC-22: 공개 페이지 동기화
**Given**: 관리자가 로드맵을 발행하면
**When**: 공개/비공개 체크박스를 활성화하면
**Then**:
- [ ] 공개 로드맵 페이지 (`/roadmap`)에 즉시 표시됨
- [ ] 비공개 설정 시 공개 페이지에서 숨김
- [ ] React Query 캐시 무효화 (Invalidation)

### AC-23: SEO 메타 태그
**Given**: 관리자가 블로그 포스트를 발행하면
**When**: SEO 제목과 SEO 설명을 입력하면
**Then**:
- [ ] 공개 블로그 포스트 페이지에서 메타 태그 반영됨
- [ ] Open Graph 태그 (og:title, og:description, og:image)
- [ ] Twitter Card 태그
- [ ] 빈 값일 경우 제목과 요약으로 자동 채움
